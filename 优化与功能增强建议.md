# Paper.js DevTools 优化与功能增强建议

> 基于代码审查的发散思考，涵盖架构问题、功能扩展、工程质量和优先级建议。

---

## 一、架构健壮性问题（基于代码审查发现）

这些是阅读代码时发现的具体隐患，建议优先处理：

| 问题 | 位置 | 风险 |
|------|------|------|
| **节点 ID 用索引路径编码**（`root_0_1_2`） | `packages/extension/src/inject/parse.ts` `findPaperItemById` | 场景树结构变化（增删/重排）后，ID 指向的 item 会错位，`selectedNode` 高亮到错误图元 |
| **messageId 用 `Date.now()`** | `packages/extension/src/content/index.ts` `sendToInjectScript` | 同毫秒内并发请求 ID 碰撞，响应错配 |
| **无消息超时机制** | `packages/extension/src/content/index.ts` `sendToInjectScript` | Inject 脚本未响应时 listener 永久泄漏，Panel 卡死 |
| **覆盖层坐标计算忽略 View zoom/center** | `packages/extension/src/inject/parse.ts` `getItemScreenBounds` | 仅按 canvas 尺寸缩放，未考虑 `view.zoom` 和 `view.center`，缩放/平移视图后高亮框错位 |
| **Scene 变化无防抖** | `packages/extension/src/panel/store/index.ts` `SCENE_CHANGE` 处理 | 高频触发时 `refreshSceneTree` + `refreshSelectedNode` 两次全量拉取，动画场景下卡顿 |
| **pickerClickHandler 用捕获阶段 + stopPropagation** | `packages/extension/src/inject/parse.ts` `enablePicker` | 拦截宿主应用所有 canvas 点击，可能破坏应用交互 |

### 建议方案

- 节点 ID 改用 Paper.js item 的 `_id`（内部递增 ID）或生成 UUID 挂载到 `item.data.__devtoolsId__`，建立 `id → item` 的 WeakMap 索引
- messageId 改用 `crypto.randomUUID()` 或递增计数器 + 随机后缀
- 消息加 5s 超时自动清理 listener
- 覆盖层坐标用 `view.projectToView(item.bounds.topLeft)` 等 Paper.js 原生坐标变换 API

## 二、核心调试能力增强（高价值）

### 1. 节点导航
- **面包屑导航**：属性面板顶部显示选中节点的祖先链，点击可快速跳转
- **选择历史栈**：前进/后退按钮在最近选中的节点间切换（类似浏览器历史）
- **键盘导航**：场景树支持 ↑↓ 移动、←→ 展开/折叠、Enter 选中、Esc 取消

### 2. 多选与批量操作
当前只支持单选。Paper.js 原生支持多选（`item.selected = true`），可扩展：
- Shift+点击多选、Ctrl+A 全选同类型
- 批量显隐、批量删除、批量对齐
- 属性面板显示多选时的公共属性（混合编辑）

### 3. 锁定节点
- 新增 `locked` 状态（写入 `item.data.__locked__`），锁定的节点在拾取器模式下不可选中、在树中标灰
- 类似 Figma 的图层锁定

### 4. 撤销/重做
- 属性编辑历史栈，记录每次 `updateNodeProperty` 的 oldValue
- 支持 Ctrl+Z / Ctrl+Y

## 三、可视化辅助（差异化亮点）

当前只有选中/悬停的矩形边框高亮，可以大幅增强：

| 功能 | 说明 |
|------|------|
| **路径句柄可视化** | 显示 Path 的锚点和贝塞尔控制柄（类似 Illustrator 直接选择工具），对矢量调试极有价值 |
| **全局 bounds 叠加** | 一键显示所有/同类型 item 的 bounds 框，快速发现布局问题 |
| **坐标标尺/网格** | Canvas 上叠加坐标轴和网格，标注鼠标坐标 |
| **线框模式** | 临时隐藏所有 fill，只看 stroke 轮廓 |
| **隔离模式** | 双击节点进入 isolate，其他节点半透明 |
| **取色器** | 读取 canvas 像素颜色，类似 PS 吸管 |
| **缩放/平移跟随** | 高亮框跟随 `view.zoom`/`view.center` 实时更新（修复坐标 bug 后） |
| **导出截图** | 将当前 canvas（含/不含覆盖层）导出 PNG |

## 四、场景树增强

### 编辑能力（当前只能查看/改属性，不能改结构）
- **右键菜单**：复制、删除、成组/解组、上移/下移层级
- **拖拽重排**：拖动节点调整层级或 reparent（调用 `item.insertAbove/appendTop`）
- **重命名**：双击节点名内联编辑 `item.name`
- **新建节点**：工具栏创建矩形/圆/路径/文字

### 信息密度
- **节点徽标**：Group 显示子节点数、Path 显示段数、Layer 显示图元总数
- **性能标记**：段数 >1000 的 Path 标红，提示性能问题
- **查找结果计数**：搜索时显示 "匹配 N 个节点"

## 五、属性面板扩展

### 覆盖更多 Paper.js 类型
当前 `extractItemProperties`（`packages/extension/src/inject/extra.ts`）只提取 7 个通用属性。可按 `item.className` 分类型提取：

| 类型 | 缺失的关键属性 |
|------|----------------|
| Path | `segments`、`closed`、`smooth`、`pathData`、`dashArray`、`dashOffset`、`miterLimit`、`cap`、`join` |
| PointText | `content`、`fontSize`、`fontFamily`、`fontWeight`、`justification`、`leading` |
| Raster | `source`、`size`、`resolution`、`smoothing` |
| SymbolItem | `symbol`（引用的 Symbol 定义） |
| Group | `clipped`（裁剪遮罩） |
| CompoundPath | `children`（子路径） |

### 颜色编辑增强
- 当前只支持纯色 hex。Paper.js 支持 `Gradient`、`RadialGradient`
- 渐变编辑器（色标、角度、位置）
- 颜色格式切换：hex / rgb / hsl / Paper.js Color 对象
- Alpha 通道独立滑块

### 变换编辑
- 旋转角度（`item.rotation`）可视化旋转盘
- 缩放（`item.scaling`）
- 倾斜（`item.shear`）
- Matrix 矩阵查看器（6 个分量）

### 其他
- **属性搜索**：属性多时过滤定位
- **重置按钮**：每个属性恢复默认
- **复制属性值**：点击复制到剪贴板
- **JSON 模式**：切换到原始 JSON 查看/编辑整个 item

## 六、性能监控面板

新增一个性能 Tab 或状态栏：

| 指标 | 实现方式 |
|------|----------|
| **FPS** | 代理 `view.onFrame` 或用 `requestAnimationFrame` 采样 |
| **渲染耗时** | 代理 `view.update`，记录 `performance.now()` 差值 |
| **图元总数** | 遍历 Project 统计 |
| **Path 段数 Top 10** | 找出最复杂的路径 |
| **View 更新频率** | 统计 `dispatchSceneChange` 调用次数/秒 |
| **内存估算** | 按图元类型 × 数量粗估 |

这能帮开发者定位"为什么我的 Paper.js 应用卡顿"。

## 七、代码导出与互操作

- **导出选中节点为 SVG**：`item.exportSVG()` 已有原生支持
- **导出为 JSON**：`item.exportJSON()` 原生支持
- **导出为 Paper.js 代码片段**：生成 `new paper.Path({...})` 形式的代码
- **导入 SVG**：`project.importSVG()`，快速测试图形
- **场景快照**：保存当前完整场景，对比不同时间点的差异（diff）

## 八、工程质量（偿还技术债）

| 项目 | 现状 | 建议 |
|------|------|------|
| **测试** | Vitest 已配置但 0 测试 | 优先为 `extractItemProperties`、`buildScopeTree`、`filterTree`、`detectPropertyType` 加单测 |
| **Background Script** | 空实现 | 可做：跨 tab 状态同步、扩展安装时注入检测、消息重试中继 |
| **ESLint** | extension 无配置 | 复用 example 的 eslint config，统一规范 |
| **错误处理** | try/catch 零散 | 统一 `safeSend` 包装 + 错误边界 + 用户可见的错误提示 |
| **类型安全** | `properties: Record<string, any>` | 按类型细化，用判别联合替代 any |

## 九、其他发散点

- **时间旅行调试**：录制每帧场景状态，拖动时间轴回放（类似 Redux DevTools）
- **事件监听器检查**：列出 item 上绑定的 `onMouseDown` 等回调
- **iframe / Web Worker 支持**：Paper.js 跑在 iframe 或 worker 时的调试
- **多版本兼容**：当前硬编码 0.12，新版 Paper.js（0.12+）API 变化时需要适配层
- **i18n**：中英文切换
- **深色主题**：DevTools 跟随系统主题
- **面板布局持久化**：记住 Splitter 比例、展开状态、上次选中的 Scope

## 十、优先级建议

按以下顺序推进：

1. **修坐标 bug + ID 稳定性** — 正确性基础，影响所有现有功能
2. **节点导航 + 键盘操作** — 日常使用体验
3. **右键菜单 + 拖拽编辑 + 新建/删除** — 从"查看器"升级为"编辑器"
4. **分类型属性提取** — 属性面板价值翻倍
5. **性能监控** — 差异化卖点
6. **路径句柄可视化** — 矢量调试杀手锏
7. **测试 + 错误处理** — 可持续迭代的基础
